# AGENTS.md — working on crossbean

Instructions for AI coding agents (and useful for humans too).

## What this is

crossbean is a **web notes app** with an AI knowledge graph. Notes are
markdown; connections between them are either explicit `[[wikilinks]]` (user edges)
or discovered automatically via embedding cosine similarity in a vector database
(AI edges). The UI is plain ES-module JS served statically — no bundler, no build
step in dev. Identity is **Clerk** (JWTs); data, vectors, and attachments live in
**Supabase** (Postgres + pgvector + storage), secured by Row-Level Security.
Runs on **Bun** (local dev) or **Vercel** (production) — Node is not required.

## Commands

| Task | Command |
|---|---|
| Run locally (dev) | `bun run start` (or `bun run dev` for watch mode) |
| Build config (Vercel) | `bun run build` — runs `node scripts/gen-config.mjs` to emit `ui/config.js` |

Requires `SUPABASE_URL`, `SUPABASE_PUBLISHABLE_KEY`, and `CLERK_PUBLISHABLE_KEY`
in `.env`. See [web/README.md](web/README.md) for setup.

## Code map

```
web/server.ts            thin Bun static host for ui/ + injects /config.js from env
scripts/gen-config.mjs   generates ui/config.js from env at Vercel build time
ui/index.html            single-page UI shell
ui/app.js                editor, note list, search, auth/vaults, theming
ui/api-supabase.js       data adapter → Clerk (identity) + supabase-js (data/vectors/storage)
ui/graph.js              force-directed canvas graph (custom physics)
ui/embed-worker.js       Transformers.js all-MiniLM-L6-v2 in a Web Worker
ui/styles.css            two themes: "paper" (default) and "terminal"
supabase/migrations/     schema, RLS, RPCs, storage (run in order 0001 → 0006)
supabase/functions/ocr/  edge function: image → hosted vision model → text (OCR)
vercel.json              Vercel deploy config (buildCommand, outputDirectory)
```

## Load-bearing constraints — do not "simplify" these away

1. **Embeddings run in the browser renderer, never server-side.**
   Transformers.js on WebAssembly inside `ui/embed-worker.js` embeds note
   text and POSTs the 384-d vector to Supabase. Do not move embedding to
   `web/server.ts` or a Supabase function.

2. **The adapter/config seam must be preserved.** `/config.js` (served by
   `web/server.ts` in dev, generated by `scripts/gen-config.mjs` on Vercel)
   sets `window.CB_CONFIG` and selects `ui/api-supabase.js`. Never call
   supabase-js or Clerk directly from `ui/app.js` or `ui/graph.js` — all I/O
   goes through the adapter.

3. **Web identity is Clerk; authorization is Postgres RLS, never client JS.**
   Clerk JWTs are passed to supabase-js as `accessToken`; every table is
   protected by RLS keyed on the Clerk user id (`supabase/migrations/0002` +
   `0006`). Do not add authorization logic to client-side JS.

4. **Two UI themes via `:root[data-theme="paper"|"terminal"]` CSS variables.**
   Canvas graph colors are read from CSS vars per frame — never hardcode
   colors in `ui/graph.js`.

5. **Similarity threshold default is 0.30.** all-MiniLM-L6-v2 cosines for
   related-but-distinct notes land around 0.3–0.5. Don't raise it to
   "intuitive" values like 0.7; nothing will connect.

6. **Embedding dimension is 384 (`EMBED_DIM`).** Changing models requires a
   schema migration and a full re-index.

## Conventions

- Plain JS (ES modules) in `ui/` — the UI is served raw, there is no bundler.
- External deps (`marked`, `supabase-js`, Transformers.js) load from jsDelivr
  CDN with the `/+esm` suffix; all three publishable keys (`SUPABASE_URL`,
  `SUPABASE_PUBLISHABLE_KEY`, `CLERK_PUBLISHABLE_KEY`) are intentionally
  public — RLS is the security boundary.
- The embedding model (~30 MB) downloads once on first browser visit and is
  cached by the browser thereafter.

## Testing

There is no headless unit test suite. Testing is end-to-end against a real
Supabase project. Run the app locally with `bun run start`, sign in, and
exercise the feature under test.

## Release

Deploy via **Vercel git integration**: push to `main`, Vercel runs
`node scripts/gen-config.mjs` (buildCommand), outputs `ui/` (outputDirectory),
and serves it as a static site.

Required env vars (set in Vercel project settings):

| Variable | Where to get it |
|---|---|
| `SUPABASE_URL` | Supabase → Settings → API → Project URL |
| `SUPABASE_PUBLISHABLE_KEY` | Supabase → Settings → API → publishable key |
| `CLERK_PUBLISHABLE_KEY` | Clerk → API Keys → Publishable key |

No native libs, no per-platform builds, no GitHub release workflow.
